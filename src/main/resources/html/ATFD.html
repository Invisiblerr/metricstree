Access To Foreign Data (ATFD)

Access To Foreign Data (ATFD) represents the number of external classes from which a given class accesses attributes, directly or via accessor-methods. High values may suggest that the class breaks encapsulation by relying on the internal representation of the classes it uses instead of the services they provide.

Proposed by M. Lanza, R. Marinescu in order to detect bad smells in code, see

M. Lanza, R. Marinescu. Object-Oriented Metrics in Practice. Using Software Metrics to Characterize, Evaluate, and Improve the Design of Object-Oriented Systems. Springer-Verlag Berlin Heidelberg, 2006.

Desirable value of this metric is less than 6.
